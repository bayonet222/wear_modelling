README file for FRIC2DOct. 2015To compile FRIC2D, the following files must exist within one directory:1) fric2d.c - source code2) getwords.c - Reads a line of input data into the character array line from the input file, then return the number of words on the line.3) getoption.c - Links a list of arguments to the program based on the arg_string passed in. 3) nrutil.h4) nrutil.c5) lubksb.c6) ludcmp.c7) nr.h 8) MakefileThe last 3-7 files include numerical recipes used by fric2d, which were implemented using Press et al. (1992).To compile FRIC2D on a Linux machine, type "make" into the terminal line, and press enter.If FRIC2D has compiled correctly, the following lines should be printed in the terminal:cc -Ofast -std=gnu99  -I. -DANSI   -c -o fric2d.o fric2d.ccc -Ofast -std=gnu99  -I. -DANSI   -c -o getwords.o getwords.ccc -Ofast -std=gnu99  -I. -DANSI   -c -o getoption.o getoption.ccc -Ofast -std=gnu99  -I. -DANSI   -c -o ludcmp.o ludcmp.ccc -Ofast -std=gnu99  -I. -DANSI   -c -o lubksb.o lubksb.ccc -Ofast -std=gnu99  -I. -DANSI   -c -o nrutil.o nrutil.cThe following object files and executables will be generated within the current directory:1) fric2d, fric2d.o2) getwords.o3) nrutil.o4) lubksb.o5) ludcmp.oTo run the fric2d executable, in the terminal enter ./fric2d -i input_filename.in -o output_filename.out -vWhere " input_filename.in " and "output_filename.out" are the names of the input file to read, and the output file that fric2d will write the results to. The "-i" and "-o" flags indicate that the name of the input file and output file will follow, respectively. The "-v" flag indicates "verbose" output statements in the output file.This directory should include sample input files, named "sample.out", for example, with the corresponding output file "sample.out". You may check that you have compiled and run FRIC2D correctly by running the FRIC2D executable on sample.in, writing the output file to a new output filename, and checking that your output file and the sample output file are the same. The command "diff" will check whether two files are identical in the terminal window, on Linux machines.CitationPress, W., S. A. Teukolsky, W. T. Vetterling, B. Flannery, (1992).
